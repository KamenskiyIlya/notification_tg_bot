# Бот для оповещений о проверке работ Devman

Этот скрипт отслеживает статус проверки Ваших работ на платформе Devman и отправляет уведомления в Telegram при появлении нового результата.

## Что необходимо подготовить

Для работы скрипта потребуется Python 3. Установите зависимости командой:

`
pip install -r requirements.txt
`

Понадобятся 2 токена и ID чата:
- Токен Devman - позволяет отправлять сайту Devman авторизованные запросы. Получить его можно в настройках профиля по этой [ссылке](https://dvmn.org/api/docs/) (необходимо авторизоваться на сайте).
- Токен бота Telegram - создайте бота через [@BotFather](https://t.me/BotFather) и получите его токен.
- ID чата - бот будет отправлять оповещения Вам, поэтому нужен Ваш ID в телеграме. Узнать его можно либо обратившись к специальному боту [@userinfobot](https://t.me/userinfobot), либо в продвинутых настройках телеграм включить отображение id чатов и аккаунтов:
<img width="482" height="518" alt="image" src="https://github.com/user-attachments/assets/691dd6a9-6063-4ad2-8e2e-be6fe4f983a6" />

Необходимо также настроить переменное окружение, для этого в папке со скриптом создайте файл с названием `.env` и пропишите в нем следующие переменные в таком виде:

- DEVMAN_TOKEN='ваш_токен_devman'
- TG_BOT_TOKEN='токен_вашего_бота'
- CHAT_ID='ID_чата_или_аккаунта'

Дополнительные параметры `.env`в для более детальной настройки:

- REQUESTS_DELAY=Время задержки перед повторными запросами(секунды)
> Если сайт не отвечает, помогает не перегружать систему или сервер, где запущен скрипт.

У скрипта имеется возможность подключения телеграм бота для отправки логов, для этого в `.env` необходимо указать следующие параметры:

- LOG_BOT_TOKEN='токен_вашего_бота'
- ADMIN_CHAT_ID='ID_чата_или_аккаунта' - куда будут отправлять все логи по работе скрипта.

## Как пользоваться

Запустите скрипт командой:

`
python main.py
`
Если Вы увидите текст как на фото, значит скрипт запустился и работает в штатном режиме:

<img width="1090" height="101" alt="image" src="https://github.com/user-attachments/assets/53f20b2c-ef14-4423-9226-3e1911ee56f4" />

После запуска скрипт начнет запрашивать сайт Devman в режиме long poling на наличие результатов проверки.
При появлении результата скрипт через бота отправит Вам оповещение в telegram с результатом проверки в следующем формате:

<img width="613" height="301" alt="image" src="https://github.com/user-attachments/assets/f0cc70db-9aa5-4084-ab03-e0a12bfbb033" />

Скрипт работает непрерывно до тех пор, пока Вы не закроете командную строку, не выключите компьютер или не нажмете комбинацию клавиш Ctrl+C.

## Логгирование

В проекте предусмотренно сохранение логов по работе скрипта. Отлавливается вся информация начиная с уровня DEBUG, выводится в консоль, а также сохраняется в файл `tg_bot.log`, который создается в директории проекта. 

<img width="1049" height="176" alt="image" src="https://github.com/user-attachments/assets/41a4512b-a376-41bf-a349-d27e7d71bcb9" />

Также имеется дополнительная возможность настроить отправку логов в реальном времени через telegram бота.
Для этого необходимо в переменном окружении указать `LOG_BOT_TOKEN` и `ADMIN_CHAT_ID`. Благодаря этому Вы сможете мониторить состояние бота с оповещениями в реальном времени, и если произойдет какая-то ошибка узнать об этом максимально быстро.
